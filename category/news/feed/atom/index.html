There are several aspects which are laid out in this blog&nbsp;post.</p> <h2>Xradar</h2> <p>In the upcoming version 1.19 wradlib will use <a href="https://docs.openradarscience.org/projects/xradar">xradar</a> for reading/writing radar data. For this to happen, I&#8217;ve ported all related code from wradlib to xradar. Xradar, a package dedicated to read and write radar data based on xarray structures, was launched in late August 2022, on initiative of Max Grover and&nbsp;me.</p> <p>A compatibility layer is kept in wradlib which enables users to continue using their code and early adopt full xradar&nbsp;dependency.</p> <h2>wradlib&nbsp;2.0</h2> <p>After the above switch to <a href="https://docs.openradarscience.org/projects/xradar">xradar</a> in wradlib 1.19 there will be quite some new code based on xarray data structures flowing into wradlib. We&#8217;ll have to cut off some old braids, too. But, using the Scientific Python stack with all those great packages, like Xarray, Dask, matplotlib, Jupyter, NumPy, SciPy, to name only a few, wradlib is still standing on the shoulders of&nbsp;giants.</p> <h2>wradlib-data</h2> <p>The <a href="https://github.com/wradlib/wradlib-data">wradlib-data</a> repository is on the move to use pooch for retrieving datasets. That way you only have to install shallow package and the data will be fetched from the repo on&nbsp;demand. </p> <h2>wradlib-notebooks</h2> <p>The <a href="https://github.com/wradlib/wradlib-notebooks">wradlib-notebooks</a> have been updated and restructured a bit to be in line with the latest developments. There is a bit of work ahead to get all notebooks up-to-date using recent&nbsp;technology.</p> <h2>wradlib-docs</h2> <p>The split of wradlib into three repos back in early 2016 was no easy decision. It helped a lot in gaining as much as possible from the emerging <a href="https://github.com">GitHub</a> cloud system. Now, as GitHub has tremendously advanced - huge thanks to GitHub for providing this all for free for us open source projects - it has come the time to get the <a href="https://github.com/wradlib/wradlib-docs">wradlib-docs</a> back into the wradlib main repository. We can much more easily keep everything together without any loose ends. This will most likely happen just after wradlib 1.19 release. In the same time the docs will have a refresh as well. We will also see a new theme by&nbsp;then.</p> <h2>wradlib-users&nbsp;forum</h2> <p>We&#8217;ve seen any kind of questions and help in our forum over the years. I always appreciated the constructive and friendly discussions. Within the <a href="https://openradarscience.org/">openradar community</a> we&#8217;ve discussed how to get more close together with respect to help users and discuss any matters of weather radar and more. Our not so new <a href="https://openradar.discourse.group/">openradar-discourse</a> discussion-group is our new location where users, developers, scientists and other weather radar enthusiasts can come together. The wradlib-users forum will go out of service later this&nbsp;year. </p> <p>I hope to see you around - Stay
